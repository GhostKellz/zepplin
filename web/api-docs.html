Overview

The Zepplin API is a RESTful interface for interacting with the package registry. All endpoints return JSON responses and use standard HTTP status codes.

Base URL: https://your-registry.example.com/api/v1

Authentication: Some endpoints require authentication via Bearer token in the Authorization header.

Packages API

GET /api/v1/packages

List all available packages in the registry.

Query Parameters

page integer Page number for pagination (default: 1)
limit integer Results per page (default: 20, max: 100)

Example Response

{ "packages": [ { "name": "zig-clap", "description": "Command line argument parsing library", "version": "0.5.0", "stars": 823 } ], "total": 156, "page": 1 }
GET /api/v1/packages/{owner}/{repo}

Get details for a specific package.

Path Parameters

owner string Package owner/organization required
repo string Package repository name required
POST /api/v1/packages/{owner}/{repo}/releases

Publish a new package release. Requires authentication.

Headers

Authorization string Bearer token required
Content-Type string multipart/form-data required

Form Fields

file file Package tarball (.tar.gz) required
tag_name string Version tag (e.g., "v1.0.0") required
body string Release notes/description
GET /api/v1/packages/{owner}/{repo}/tarball/{version}

Download a package tarball for a specific version.

GET /api/v1/packages/{owner}/{repo}/hash/{version}

Get the SHA256 hash for a package version (used by zig fetch).

Search API

GET /api/v1/search

Search for packages by name, description, or tags.

Query Parameters

q string Search query required
category string Filter by category
limit integer Max results (default: 20)
GET /api/v1/trending

Get trending packages based on recent activity and popularity.

Authentication API

GET /api/v1/auth/oauth/github/login

Initiate GitHub OAuth flow. Redirects to GitHub for authentication.

GET /api/v1/auth/oidc/microsoft/login

Initiate Microsoft Entra (Azure AD) OIDC flow.

POST /api/v1/auth/register

Register a new user account.

Request Body (JSON)

username string Unique username required
email string Email address required
password string Password (min 8 chars) required

Health Check

GET /health

Check if the registry server is healthy and responding.

Example Response

{ "status": "healthy", "version": "0.5.0" }
