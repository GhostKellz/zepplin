Getting Started with Zepplin

Set up Zepplin and start managing your Zig dependencies in minutes

1Prerequisites

Before you begin, ensure you have the following installed:

  • Zig 0.13.0 or later - Download Zig
  • Git - For fetching packages from repositories
  • A terminal/command prompt

2Adding Dependencies

Option A: Using zig fetch (Recommended)

The easiest way to add a dependency is with zig fetch --save:

# Add latest from main branch zig fetch --save https://github.com/user/package/archive/main.tar.gz # Or pin to a specific version tag (recommended for stability) zig fetch --save https://github.com/user/package/archive/refs/tags/v1.0.0.tar.gz

This automatically adds the dependency with its hash to your build.zig.zon.

Option B: Manual build.zig.zon Entry

You can also manually add dependencies to your build.zig.zon:

.dependencies = .{ .zqlite = .{ .url = "https://github.com/ghostkellz/zqlite/archive/refs/tags/v1.5.4.tar.gz", .hash = "1220abc123...", // Get this from zig fetch output }, },

Tip: Always pin to a version tag (e.g., refs/tags/v1.0.0) for production projects. Using main or master branches can lead to unexpected breaking changes.

Option C: Clone and Build from Source

# Clone the repository git clone https://github.com/ghostkellz/zepplin.git cd zepplin # Build the CLI tool zig build # The binary is in zig-out/bin/zepplin

3Initialize a Project

Create a new Zepplin project or initialize an existing Zig project:

zepplin init

This creates two files:

  • zepplin.toml - Your project configuration and dependencies
  • zepplin.lock - Lock file for reproducible builds

4Add Dependencies

Add packages to your project using the add command:

# Add a package (latest version) zepplin add zig-clap # Add a specific version zepplin add zap@0.3.0 # Add from a custom registry zepplin add mypackage --registry https://my-registry.example.com

5Build Your Project

Build your project with all dependencies resolved:

zepplin build

This command resolves dependencies, downloads packages, and runs zig build.

Tip: Discover Packages

Use the discover command to search for packages:

zepplin discover "http server" zepplin trending zepplin browse --category=cli

6Publishing Packages

Ready to share your package? Make sure your build.zig.zon has the required fields:

.name = "my-awesome-package", .version = "1.0.0", .description = "A brief description of your package", .license = "MIT",

Then publish:

# Login to the registry (first time only) zepplin login # Publish your package zepplin publish
