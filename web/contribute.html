Development Setup

Prerequisites

  • Zig 0.16.0+ - Required for building Zepplin
  • Git - For cloning and version control
  • SQLite - Database backend (via zqlite)

Clone and Build

# Clone the repository git clone https://github.com/ghostkellz/zepplin.git cd zepplin # Build the project zig build # Run tests zig build test # Start the development server ./zig-out/bin/zepplin serve 8080

Project Structure

zepplin/ ├── src/ │ ├── main.zig # Entry point │ ├── cli/ # CLI commands │ ├── server/ # HTTP server │ ├── database/ # Database layer │ ├── auth/ # Authentication │ ├── storage/ # Package storage │ └── zigistry/ # Zigistry client ├── web/ # Static web files ├── docs/ # Documentation └── build.zig # Build configuration

Submitting Changes

Pull Request Process

  1. Fork the repository on GitHub
  2. Create a branch for your changes: git checkout -b feature/my-feature
  3. Make changes following our code style
  4. Test your changes: zig build test
  5. Commit with clear, descriptive messages
  6. Push to your fork and open a Pull Request

Code Style

  • Follow Zig's standard style guidelines
  • Use zig fmt to format code before committing
  • Write tests for new functionality
  • Keep commits focused and atomic
  • Write clear commit messages explaining the "why"
Commit Message Format

Use conventional commits when possible:

feat: add package search filtering fix: resolve authentication timeout issue docs: update CLI reference for new commands refactor: simplify database connection handling

Reporting Issues

When reporting bugs, please include:

  • Zig version (zig version)
  • Operating system and architecture
  • Steps to reproduce the issue
  • Expected vs actual behavior
  • Relevant error messages or logs
